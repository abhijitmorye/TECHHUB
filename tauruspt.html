Performance Testing with Taurus

In this article, we will understand how we can do performance testing using Taurus which is an automation tools.

Topics :

  • What is Taurus?
  • Installation and Upgrades
  • Taurus Hands on Tutorial

What is Taurus?

  • Taurus, an open source test automation tool that extends and abstracts JMeter and other testing tools, and helps to overcome different challenges such as Automation and integration with other systems.
  • Taurus provides a simple way to create, run and analyze performance tests.
  • Taurus is an acronym that stands for Test Automation Running Smoothly, which reflects its baseline mission - making the process of test configuration and execution as simple as possible.

Difficulties with different Testing tools :

    Different open source tools are available in market i.e. Jmeter, Gatling, Locust.io.All these tools come with their advantages and disadvantages.

  • Different framework and underlying technologies(Jmeter based on Java and Gatling based on Scala).
  • Different methods to create scenarios.
  • Different Functions.
  • Different automation techniques with CI/CD tools.

Idea Behind Taurus

Taurus takes the advantages of underlying open source perfromance testing tools such as Jmeter,Locust.io,Gatling etc.It provides the simple domain specific language i.e YAML,using which we can create performance script and can specify which tool/executer we want to use for performance testing.

As you can see from above picture,inner most box contains different open source tools that are used as execution engine by Taurus.Taurus uses Jmeter as it's default execution engine,however we can also specify which execution engine we want to use for testing.

Taurus can be scaled using Blazemeter cloud solution.

Advantages using Taurus

  1. Extremely simple setup and upgrading
  2. Platform-independency - runs where Python and Java can run
  3. An easily-readable, version control friendly and unified DSL (Domain-specific language) to define load test scenarios
  4. The ability to execute existing JMeter (or Grinder or Gatling or Selenium) tests
  5. The ability to create new tests from scratch using the aforementioned DSL
  6. The ability to merge multiple existing test scripts and/or DSL-driven tests into a single scenario
  7. Real-time reporting
  8. Multiple test output formats (HTML,XML,JTL)
  9. Integration with the blazemeter.com reporting service
  10. An easy way to define flexible pass/fail criteria. In case the results exceed a threshold, it is possible to automatically mark test(s) as failed
